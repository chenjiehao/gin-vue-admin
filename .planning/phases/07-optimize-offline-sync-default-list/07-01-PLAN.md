## Goal

确保访问"数据集成 > 离线同步"二级菜单时，默认显示任务列表页面（offlineSync.vue），而不是任务表单页面（offlineSyncForm.vue）。

## Problem Statement

根据测试反馈，当用户访问"数据集成 > 离线同步"菜单时，页面可能显示的是 offlineSyncForm（任务表单）而不是 offlineSync（任务列表）。这会导致用户无法看到现有的任务列表。

## Root Cause

问题出在 `web/src/permission.js` 的 `addRouteByChildren` 函数中。当父路由有子路由时，代码会设置重定向到"第一个子路由"，但没有检查子路由是否被标记为 hidden。

对于 offlineSync 菜单：
- 它有一个子路由 OfflineSyncForm (hidden=1)
- 原来的代码会设置 redirect 到 OfflineSyncForm
- 导致访问 offlineSync 时被重定向到 OfflineSyncForm

## Tasks

### Task 1: 检查菜单配置
- [x] 检查数据库 sys_base_menus 表中离线同步相关菜单
- [x] 验证 offlineSync 菜单是否有子路由指向 offlineSyncForm
- [x] 确认 OfflineSyncForm 被标记为 hidden=1

### Task 2: 修复路由注册逻辑
- [x] 修改 `web/src/permission.js` 中的 `addRouteByChildren` 函数
- [x] 只选择第一个可见的子菜单（!child.hidden）进行重定向
- [x] 如果没有可见子菜单，父路由仍然注册但不设置重定向

### Task 3: 验证修复
- [ ] 测试访问离线同步菜单，确认默认显示任务列表

## Fix Applied

```javascript
// Before (bug):
const firstChild = route.children[0]  // 任意第一个子路由，包括 hidden
if (firstChild) {
   // redirect to first child...
}

// After (fixed):
const firstVisibleChild = route.children.find(child => !child.hidden)  // 只选可见子路由
if (firstVisibleChild) {
   // redirect to first visible child...
}
// 父路由始终注册，但没有可见子路由时不设置重定向
router.addRoute('layout', parentRoute)
```

## Files Modified

- `web/src/permission.js` - 修复了 addRouteByChildren 函数中的重定向逻辑

访问"数据集成 > 离线同步"时，直接显示任务列表，包含6条模拟任务数据，状态标签显示正确。
