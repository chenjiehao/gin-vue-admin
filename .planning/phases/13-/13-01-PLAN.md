## Context

Phase 8-11 完成了数据源管理的 CRUD 和离线同步基础 UI。
Phase 12 完成了连接测试和批量操作。

当前离线同步任务配置时，只能选择数据源但无法选择具体的表。
本阶段新增"根据数据源获取表列表"能力，为离线同步的表级配置做准备。

## Approach

**技术方案：**
- 复用 Phase 12 连接测试的驱动连接能力
- 新增 `DataSourceService.GetTables()` 方法
- 查询表的 SQL 因数据库类型而异：
  - MySQL: `SHOW TABLES FROM {database}`
  - PostgreSQL: `SELECT tablename FROM pg_tables WHERE schemaname = 'public'`
  - SQLServer: `SELECT name FROM sys.tables`
  - 其他国产数据库暂返回"不支持"

**API 设计：**
```
GET /dataSource/getTables?dataSourceId=1
Response: { code: 0, data: { tables: ["user", "order", "product"] }, msg: "success" }
```

## Tasks

1. **DataSourceService.GetTables** — 根据数据源类型和连接信息查询表列表
2. **GetTables API Handler** — 接收 dataSourceId，返回表名数组
3. **路由注册 + 权限配置** — Casbin 策略 + API 信息录入
