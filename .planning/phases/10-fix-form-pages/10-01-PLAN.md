## Goal

修复"新增数据源"和"新增离线同步任务"按钮点击后页面无法访问的问题。

## Root Cause

Phase 09 删除了 `OfflineSyncForm` (id=14) 和 `dataSourceForm` (id=16) 这两个隐藏菜单记录。

但这两个表单页**本身**是通过 `router.push({ name: 'OfflineSyncForm' })` 和 `router.push({ name: 'dataSourceForm' })` 访问的，不是菜单导航。

问题在于：
1. GVA 的路由来自 `sys_base_menus` 表（通过 `SetAsyncRouter` 获取）
2. 这两个表单页从未作为独立路由注册过 —— 它们之前能work纯粹是因为它们"恰好"作为 hidden=1 的菜单记录存在
3. 删除后，这些路由名不再存在于系统中
4. `router.push({ name: 'OfflineSyncForm' })` 抛出 "No match for" 错误

## Tasks

### Task 1: 注册表单页路由

<read_first>
- web/src/router/index.js — 静态路由配置
- web/src/pathInfo.json — 组件→路由名映射
</read_first>

<acceptance_criteria>
- 点击"新增"按钮后，表单页能正常打开（不是404）
- router.hasRoute('OfflineSyncForm') === true
- router.hasRoute('dataSourceForm') === true
</acceptance_criteria>

<action>
在 `web/src/router/index.js` 中添加静态路由注册：

```javascript
{
  path: '/dataIntegration/offlineSyncForm',
  name: 'OfflineSyncForm',
  component: () => import('@/view/dataIntegration/offlineSyncForm.vue')
},
{
  path: '/dataIntegration/dataSourceForm',
  name: 'dataSourceForm',
  component: () => import('@/view/dataIntegration/dataSourceForm.vue')
},
```

注意：
- pathInfo.json 中的映射保持不变（用于后端返回路由时的动态解析）
- 这里使用 @/ 别名路径（相对于 web/src/）
</action>
