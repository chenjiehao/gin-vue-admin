## Context

当前离线同步表单：
- 选择数据源 → 直接选表

问题：表是全局的，但实际需要指定 database 才能准确查表。

优化后：
- 选择数据源 → 自动填充 database（只读）→ 加载表列表

数据源的 database 字段已存储了目标库名称，前端直接引用即可。

## Approach

**后端修改：**
- GetTables API 增加 `database` 参数
- MySQL: `USE database; SHOW TABLES;`
- PostgreSQL: 连接时 dbname 参数化

**前端修改：**
- 数据源下拉选择后，自动用 dataSource.database 填充 database 下拉
- database 下拉禁用（不可编辑，只读展示）
- 表下拉依赖 database，切换 database 时清空已选表
